Why redesign?

v1 — Helm chart
  • All logic hidden in _helpers.tpl — type inference, deep merges
  • Rendered output ≠ what you read in Git
  • Rate limiter injected as an inline .java file
  • ApplicationSet + child app per route — extra controllers
v2 — Kamelets + Pipes
  • Operator-native abstraction — no templating layer at all
  • What's in Git is what hits the cluster
  • Rate limiter = versioned jar in the Maven mirror
  • One directory-sync Argo app; prune + self-heal per file

A route is one small Pipe file

# routes/orders-kk.yaml — the WHOLE route
kind: Pipe
metadata: { name: orders-kk }
spec:
  source:
    ref: { kind: Kamelet, name: kafka-scram-source }
    properties: { topic: orders-in }
  steps:
    - ref: { kind: Kamelet, name: rate-limit-action }
      properties: { key: orders, rate: 10, burst: 20 }
  sink:
    ref: { kind: Kamelet, name: kafka-scram-sink }
    properties: { topic: orders-out }

5 Kamelets = the vocabulary (written once)

  • kafka-scram-source / -sink — brokers, SASL, secrets, per-side overrides
  • mtls-http-source — platform-http :8443, client-auth REQUIRED
  • mtls-http-sink — camel:http + client keystore, pinned truststore
  • rate-limit-action — Redis token bucket via ratelimit-plugin.jar

Route "type" = which Kamelets it composes. No enum, no inference — nothing to drift.

Metrics — external Prometheus, no new firewall rule

  • Prometheus trait on by default (central trait config) — every route exposes /q/metrics, authors never think about it
  • Prometheus discovers pods with kubernetes_sd and scrapes through the kube-API proxy on :6443 — the same opening Argo uses
  • Scoped prometheus-scraper SA: list pods + pods/proxy in camel-k, nothing else
  • New route ⇒ appears in Prometheus automatically, labeled route=<name>
# scrape path per pod

      <pod>:8080/proxy/q/metrics

Rate limiting — correct across replicas, routes, restarts

  • Token bucket state in Redis, refill+consume as one atomic Lua script
  • block mode (Kafka) = native backpressure · reject mode (HTTP) = 429 + Retry-After
  • Same key on two Pipes ⇒ one shared global SLA across routes
  • In-memory limiters silently multiply the SLA per replica — Redis is the one value every pod agrees on
v2 packaging

The Java bean ships as ratelimit-plugin.jar in the Maven mirror. The rate-limit-action Kamelet declares it as a dependency — upgrading the limiter for every route is a one-line version bump in one file.
